SQL 语句逻辑执行顺序

Posted on 2018-09-21 | Edited on 2018-11-29 | In 数据库 | Views:

SQL语句准备执行阶段

SQL执行顺序

from 子句

from 子句执行顺序为从后往前、从右到左，写在最后的表（也被称为驱动表）将被最先处理，下面是一些不错的建议：

  • 在 from 子句中包含多个表的情况下，你必须选择记录条数最少的表作为基础表；
  • 如果有 3 个以上的表连接查询, 那就需要选择交叉表(intersection table)作为驱动表，交叉表是指被其他表所引用的表；
  • 多表连接时，使用表的别名并把别名前缀于每个列上。可以减少解析的时间并减少那些由列名歧义引起的语法错误。

where 子句

where 子句执行顺序为自下而上、从右到左，根据这个原理，表之间的连接必须写在其他 where 条件之前，可以过滤掉最大数量记录的条件必须写在 where 子句的末尾。

group by 子句

group by 执行顺序从左往右分组，为了提高 group by 语句的效率，可以通过将不需要的记录在 group by 之前过滤掉，即在 group by 前使用 where 来过虑，而尽量避免 group by 后再 having 过滤。

having 子句

having 是对检索的结果集进行过滤，应尽量少用此子句，因为它较耗资源。

select 子句

在书写 select 子句时应少用星号，尽量取字段名称， 因为在解析的过程中, 会将星号转换成所有的列名，这个工作是通过查询数据字典完成的，使用列名意味着将减少消耗时间。

order by 子句

order by 子句执行顺序为从左到右排序，很耗资源。
