Elasticsearch Client 使用介绍

Posted on 2018-09-30 | In elasticsearch | Views:

基本概念

Elasticsearch Client 指的是 Elasticsearch 的 Java API 所提供的客户端，我们以多种方式使用 Client：

  • 在已有的集群上执行标准的 index、get、delete、search 操作
  • 在正在运行的集群上执行管理任务

获取 Elasticsearch Client 很简单，最常用的方式是创建TransportClient实例。TransportClient 使用 transport 模块远程连接到 Elasticsearch 集群，它自身不加入集群，而是简单地获取一个或多个初始传输地址，并在每个操作上以轮询的方式与集群通信。

需要注意的是官方文档中说明 6.x 及其以下版本可以正常使用 TransportClient，在 7.x 会将其废弃，而到 8.x 则会彻底废除，因此在使用需要注意 Elasticsearch 客户端 API 的版本以及后续的迁移计划。

实践示例

一个简单的创建 TransportClient 的例子：

TransportClient client = new PreBuiltTransportClient(Settings.EMPTY)
        .addTransportAddress(new TransportAddress(InetAddress.getByName("127.0.0.1"),9300));

// do something...

client.close();

上面的例子中，我们使用了默认的配置，它假设连接的目标集群的名称为elasticsearch，如果需要使用自定义的集群名称，则须显示设置：

Settings settings = Settings.builder()
        .put("cluster.name", "myClusterName").build();
TransportClient client = new PreBuiltTransportClient(settings);

如果无法知道确定所使用的名称，也可以设置忽略集群名称：

Settings settings = Settings.builder()
        .put("client.transport.ignore_cluster_name",true).build();
TransportClient client = new PreBuiltTransportClient(settings);

常用的客户端级别的配置项汇总：

参数名 说明
cluster.name 目标集群名称
client.transport.ignore_cluster_name 忽略目标集群名称
client.transport.sniff 客户端是否自动嗅探集群节点
client.transport.ping_timeout 客户端 ping 集群节点并等待其响应的超时时间，默认为 5 秒，结合client.transport.sniff使用
client.transport.nodes_sampler_interval 对列出和连接的节点进行 ping 采样的频率，默认为 5 秒，结合client.transport.sniff使用
